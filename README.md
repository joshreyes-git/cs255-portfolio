# cs255-portfolio

1. Briefly summarize the DriverPass project. Who was the client? What type of system did they want?
DriverPass was a system designed for a driving school business that wanted to help students prepare for their driving tests. The client needed a web-based system that allows users to register, schedule lessons, take practice tests, and track progress.

2. What did you do particularly well?
I did well at organizing the system design and clearly showing how different parts of the system connect using UML diagrams. I also made sure the system covered the main user needs.

3. If you could revise one part, what would it be and how would you improve it?
I would improve my diagrams by adding more detail and making them cleaner and easier to read. I would also include more real-world scenarios to make the design stronger.

4. How did you interpret the user’s needs and implement them into your design? Why is this important?
I focused on the main tasks users needed, like scheduling lessons and taking practice tests, and built the system around those features. This is important because the system needs to actually solve the user’s problems and be easy to use.

5. How do you approach designing software? What will you do in the future?
I start by understanding the requirements, then break the system into smaller parts and design each one. In the future, I will focus more on planning, feedback, and improving my diagrams to make my designs clearer.
